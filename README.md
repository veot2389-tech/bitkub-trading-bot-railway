## 🧠 2. อัลกอริทึมและเทคนิคคณิตศาสตร์ (Modern Quantitative Trading)

บอทตัวนี้ทำงานบนหลักการ **Quantitative Analysis** 3 เสาหลัก:

### A. Dynamic Fibonacci Grid (ระยะห่างมหัศจรรย์)
บอทไม่ได้ซื้อตามระยะคงที่ แต่ใช้ **ลำดับเลขฟีโบนักชี** มาเป็นตัวคูณระยะห่าง:
- **สูตร:** `Base Step * Fibonacci_Multiplier[Layer]`
- **เทคนิค:** ไม้ 1-2 จะเข้าใกล้ราคาปัจจุบัน (1:1) เพื่อเก็บรอบไว ไม้ 3-5 จะขยายระยะห่างออกไป (2x, 3x, 5x) เพื่อ "ดักราคา" ในจุดที่ปลอดภัยที่สุด ช่วยลดการใช้เงินสดฟุ่มเฟือยในจุดที่เสี่ยง

### B. Probability Filters (ตัวกรองความน่าจะเป็น)
ป้องกันการซื้อในจุดที่เสียเปรียบด้วยสถิติ:
1.  **Z-Score Monitor:** คำนวณว่าราคา ณ ปัจจุบันอยู่ห่างจากค่าเฉลี่ยกี่เท่าของส่วนเบี่ยงเบนมาตรฐาน (Std Dev)
    *   ถ้า `Z-Score > 0.8`: แปลว่าราคาพุ่งแรงเกินไป "ไม่ซื้อ" ป้องกันการดอยยอด
2.  **Momentum Slope:** ตรวจสอบความชันของราคา 3 จุดล่าสุด
    *   ถ้าเป็นขาลงรุนแรงเกิน `-1.5 * Std Dev`: บอทจะมองว่าเป็น "มีดร่วง" และจะรอนิ่งๆ จนกว่าราคาจะเริ่มเสถียร

### C. Advanced Dynamic Slot Management (การบริหารหน้าตักอัจฉริยะ)
บอทจะคำนวณจาก **Total Equity (เงินสด + มูลค่าเหรียญรวม)** และนำมาหารด้วย **จำนวน Slot ทั้งหมด ( coins * levels)**
- **ประโยชน์:** ทำให้ขนาดไม้ซื้อ (`dynamic_amt`) มีความเสถียร ไม่ขยับขึ้นลงตามจำนวนไม้ที่ถืออยู่ และจะเติบโตขึ้นเรื่อยๆ ตามขนาดพอร์ตที่ใหญ่ขึ้น (Compounding Effect)
- **การตั้งค่า:** สามารถปรับ `max_amount_per_layer` และ `budget_utilization_pct` ได้ใน `config.json`

## ⚓ 3. ระบบความเสถียร (Engineering Stability)

### 🔌 ระบบกู้คืนสถานะ (Self-Healing)
- **Power Failure:** หากไฟดับ ข้อมูลไม้ที่ถืออยู่จะอยู่ใน `trading_v8.db` เมื่อเปิดเครื่องใหม่ บอทจะดึงมูลค่าเหรียญจริงจากบัญชีมาเทียบ (Balance Sync) เพื่อตรวจสอบว่าของยังอยู่ครบไหม
- **WebSocket Reconnect:** มีวงจรวนลูป `try-except` ชนิดพิเศษที่จะพยายามต่อท่อข้อมูลกับ Bitkub ใหม่ทุกๆ 5 วินาทีหากเน็ตหลุด

### 🧵 Multi-Threading Architecture
บอททำงานแบบ 3 ห้องปฏิบัติการแยกออกจากกัน:
1.  **Main Loop (Asyncio):** รัน Logic การซื้อขายและเฝ้าราคา
2.  **Telegram Thread:** แยกออกมาต่างหากเพื่อให้การตอบสนองปุ่มกด "ลื่นไหล" ไม่ค้าง
3.  **Flask API Thread:** รันเว็บเซิร์ฟเวอร์เล็กๆ พอร์ต 5000 สำหรับรายงานสถานะแบบ JSON

## 💰 4. วิศวกรรมกำไร (The 0.6% Net Rule)

บอทเวอร์ชัน 8.8 ถูกโปรแกรมให้ **"เห็นแก่ประโยชน์ของผู้ใช้"** เป็นที่หนึ่ง:
- **Gross Profit:** กำไรบนหน้าจอ
- **Net Profit:** กำไรที่หักค่าธรรมเนียม Bitkub (0.25% x 2 = 0.5%) แล้ว
- **กฎเหล็ก:** บอทจะ "ไม่ยอมขาย" หากกำไรสุทธิไม่ถึง **0.6%** (หรือประมาณ 1.1% บนหน้าจอ) เพื่อให้คุณมั่นใจว่าทุกครั้งที่มีการขยับพอร์ต คุณรวยขึ้นจริงๆ

## 🛠️ 5. คู่มือคำสั่งคุมบอท (Operator's Command List)

ใช้คำสั่งเหล่านี้ใน Ubuntu/WSL เมื่อต้องการตรวจสอบระบบอย่างละเอียด:

| งานที่ต้องการ | คำสั่งที่ใช้ |

| **ตรวจเช็คโปรเซสค้าง** | `ps aux | grep trading_bot_v8.py` |

## 🧪 6. คำแนะนำการแก้ปัญหา (Troubleshooting)

- **ทำไมบอทเงียบ?**: เช็ค Z-Score ใน Telegram ถ้าค่าเป็นบวก (Z > 0) คือราคายังแพงไป บอทกำลังดักรอจุดที่ถูกกว่านี้
- **ทำไม Next Buy ขยับ?**: เพราะราคามันเปลี่ยนไปตลอดเวลา บอทจะใช้การคำนวณฐานราคาที่ต่ำที่สุดของไม้ที่มีอยู่มาหาจุดซื้อใหม่เสมอ
- **WS Error?**: ปกติมาก บอทจะ Restart ตัวเองและเชื่อมต่อใหม่ให้เองภายใน 5 วินาที
