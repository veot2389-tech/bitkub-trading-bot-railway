# 🚀 CONTINUOUS DCA GRID (Render Quant)
*บอทเทรดคริปโตบน Bitkub อัตโนมัติด้วยระบบ ถัวดักซื้อแบบเบาๆ ลากยาวๆ (Continuous DCA Grid)*

---

## 📖 สรุปภาพรวมหลักการทำงานของบอท
บอทตัวนี้ถูกออกแบบตามความต้องการที่ว่า **"ให้มีไม้ดักซื้อไปเรื่อยๆ ยาวๆ ไม่เงินหมดไว!"** ดังนั้นเราจึงเปลี่ยนจากการทุ่มเงินด้วยฟีโบนักชีแบบบ้าคลั่ง มาเป็นระบบ **สายเอาตัวรอด (DCA แบบอ่อนโยน)** 

### ⚙️ 2 หัวใจหลักที่ทำให้บอทรองรับหลายไม้ได้ดั่งใจ:

#### 1. ระยะห่างของแต่ละไม้เปลี่ยนไปตาม "สวิงของตลาด" (Dynamic Grid)
บอทจะไม่กำหนดช่องว่างการซื้อขายแบบตายตัว (เช่น คงที่ที่ 1% ตลอด) แต่บอทคอยเช็ค **"ความผันผวน 30 ไม้ล่าสุด"** ตลอดเวลา
- **ถ้ากราฟนิ่ง:** บอทจะใช้ความห่างเริ่มต้นที่ `0.5%`
- **ถ้ากราฟกระชากแรง:** บอทจะถ่างระยะการซื้อออกไปตามความแกว่งของตลาดแบบอัตโนมัติ

#### 2. เพิ่มตัวคูณอย่างค่อยเป็นค่อยไป (Scale-up DCA)
ถ้าคุณติดดอยลึกขึ้น บอทจะใช้เงินรับของตามสัดส่วนนี้:
`[1x, 1.1x, 1.2x, 1.3x, 1.4x, 1.5x, 1.6x, 1.8x, 2.0x, ... ]`
โดยมี **Base Unit เริ่มต้นแค่ 1.2% ของเงินทุนทั้งหมด!!** (สมมติมี 1000 บาท บอทจะเริ่มรับไม้แรกแค่ 12 บาท)

*อธิบายให้เห็นภาพ:*
- ด้วยขนาดเริ่มต้นที่เล็กจิ๋ว และตัวคูณที่ค่อยเป็นค่อยไป (ไม่กระชากกระโดดทะลุหลอดเหมือนเก่า) ถ้าตลาดเจอดาวน์เทรนหนักๆ **เงินพอร์ตของคุณจะยืดอายุรองรับการดักซื้อไปได้เรื่อยๆ มากกว่า 30-50 ไม้** โดยไม่ทันหมดกระสุนกลางทาง
- ยิ่งราคาลงลึกๆ บอทก็จะไปรับซื้อหนักขึ้นในระดับก้นหลุมแบบพองาม ไม่เสี่ยงจนเกินไป ทำให้ลากต้นทุนเฉลี่ยดิ่งลงมาได้เช่นเดิม

- ไปที่แท็บ **Logs**

## 🗄️ วิธีดูประวัติการเทรดและสถิติ (Supabase Database)
บอทได้ผูกกับฐานข้อมูลให้อัตโนมัติในทุกการกระทำ ไม่ต้องรันสคริปต์ `.sql` เพื่อสร้างตารางให้วุ่นวาย
- เข้าไปที่ [Supabase Dashboard](https://supabase.com/dashboard)
- เลือกโปรเจกต์ของคุณ 
- เมนูด้านซ้าย เลือก **Table Editor**
- คุณสามารถคลิกดูตารางได้เลย เช่น:
  - `trades` : ประวัติการซื้อ/ขายและกำไรเป็นรายไม้
  - `daily_snapshots` : สรุปยอดเงินทุนและกำไรรวมของแต่ละวัน
  - `layers` : ข้อมูลไม้ที่ติดดอยอยู่ (ตรงกับที่โชว์ใน Telegram)

*(แม้ว่าหน้าเว็บ Supabase จะเผลอลบตารางทิ้งไป แต่ตัวบอทมีฟังก์ชันคืนชีพตารางให้ใหม่ทันทีที่มีการขยับครับไม่ต้องกังวล!)*
